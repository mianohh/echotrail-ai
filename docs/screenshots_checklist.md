## Screenshot Guidelines

### Technical Requirements
- **Resolution**: 1920x1080 minimum
- **Format**: PNG with high quality
- **Browser**: Chrome with clean profile, no extensions visible
- **Zoom**: 100% browser zoom level
- **Window**: Full screen browser window

### Visual Standards
- **Data**: Use demo data for realistic, compelling content
- **UI State**: Show interactive elements (hover states, active buttons)
- **Consistency**: Same browser, same user, same session
- **Lighting**: Consistent lighting/contrast across all screenshots
- **Branding**: EchoTrail AI logo and branding visible where appropriate

### Content Guidelines
- **Realistic Data**: Use the demo dataset for authentic-looking content
- **Emotional Range**: Show variety in moods, themes, and emotional tones
- **Progression**: Screenshots should tell a story from onboarding to insights
- **Polish**: Highlight premium UI elements, animations, and attention to detail
- **Value**: Each screenshot should demonstrate clear user value

### Bonus Screenshots (Optional)
- **Mobile Responsive**: Key screens on mobile viewport (375x812)
- **Dark Mode**: Same screens showing dark theme consistency
- **Empty States**: Clean empty states with helpful CTAs
- **Error Handling**: Graceful error messages and recovery flows
